# 📍 GitHub Actions 操作指南 - 点击哪里

## 🎯 触发构建的步骤

### 步骤1：点击左侧的"构建Android APK"

在左侧边栏（Actions菜单下），找到并点击：
- **"构建Android APK"** （工作流名称）

---

### 步骤2：点击"Run workflow"按钮

点击"构建Android APK"后，在页面右侧会看到：
- 一个蓝色的 **"Run workflow"** 按钮
- 点击这个按钮

---

### 步骤3：选择分支并确认

1. 会弹出一个下拉菜单
2. 选择分支：**"main"** （或 "master"）
3. 点击绿色的 **"Run workflow"** 按钮确认

---

### 步骤4：等待构建

- 构建会自动开始
- 可以在页面看到构建进度
- 首次构建需要 **10-20分钟**

---

## 🔍 详细操作

### 如果看不到"构建Android APK"

1. 检查文件是否上传成功
2. 确认 `.github/workflows/build_apk.yml` 文件存在
3. 刷新页面（F5）

### 如果点击后没有"Run workflow"按钮

1. 确认工作流文件格式正确
2. 检查是否有语法错误
3. 可能需要等待几秒钟让GitHub识别工作流

---

## ✅ 成功标志

点击"Run workflow"后，你会看到：
- 一个新的构建任务出现在列表中
- 状态显示为 🟡 黄色（正在运行）
- 可以点击查看详细日志

---

## 📱 快速操作

**最简单的方法：**

1. 左侧边栏 → 点击 **"构建Android APK"**
2. 右侧 → 点击 **"Run workflow"**
3. 选择分支 → 点击 **"Run workflow"** 确认
4. 等待构建完成

就这么简单！
